# React 렌더링 구조 정리 (sample2)

## 목차
1. 핵심 관계
2. 실행 흐름
3. 파일별 역할
4. 가상 DOM(Virtual DOM) 정리
5. Component와 Props
6. 한 줄 요약

## 1. 핵심 관계
- `index.html`: 브라우저가 처음 읽는 HTML 템플릿
- `main.tsx`: React 앱을 시작(부트스트랩)하는 진입 파일
- `App.tsx`: 실제 화면 UI를 정의하는 루트 컴포넌트

## 2. 실행 흐름
1. `index.html`이 로드된다.
2. `index.html`의 `<script type="module" src="/src/main.tsx"></script>`가 `main.tsx`를 실행한다.
3. `main.tsx`가 `#root` DOM을 찾아 React를 마운트한다.
4. `main.tsx`에서 `<App />`를 렌더링한다.
5. `App.tsx`가 반환한 JSX가 실제 화면에 보인다.

## 3. 파일별 역할

### 3-1) index.html
- React 앱이 붙을 자리 `<div id="root"></div>`를 제공한다.
- `main.tsx`를 불러오는 시작 스크립트를 가진다.

즉, 앱 컨테이너 + 엔트리 스크립트 로더 역할이다.

### 3-2) main.tsx
- `createRoot(document.getElementById('root')!)`로 루트 컨테이너를 잡는다.
- `.render(...)`로 React 컴포넌트를 화면에 렌더링한다.
- `<StrictMode>`로 감싸 개발 중 잠재 문제를 더 잘 찾게 해준다.
- `App` 컴포넌트를 import 해서 최상위 화면으로 렌더링한다.

즉, React 앱 실행/마운트 담당이다.

### 3-3) App.tsx
- 실제 화면 내용을 JSX로 정의한다.
- 현재 예제는 `<h1>Vite + React</h1>`를 렌더링한다.

즉, UI 본문 담당이다.

## 4. 가상 DOM(Virtual DOM) 정리

### 4-1) 가상 DOM이란?
- 실제 브라우저 DOM을 자바스크립트 객체 형태로 표현한 "가벼운 UI 스냅샷"이다.
- React는 상태(state)나 props가 바뀌면 새 가상 DOM 트리를 만든다.

### 4-2) 왜 쓰는가?
- 실제 DOM 조작은 상대적으로 비용이 크다.
- React는 이전 가상 DOM과 새 가상 DOM을 비교(diff)해서, 바뀐 부분만 실제 DOM에 반영한다.
- 결과적으로 불필요한 DOM 업데이트를 줄여 성능과 일관성을 확보한다.

### 4-3) 동작 순서
1. 상태/props 변경 발생
2. 새 가상 DOM 생성
3. 이전 가상 DOM과 비교(diff)
4. 변경점 계산 후 실제 DOM에 최소 변경 반영(commit)

### 4-4) 자주 하는 오해
- "가상 DOM = 항상 가장 빠름"은 아니다.
- 핵심은 무조건 속도 1등이 아니라, 선언형 UI 개발에서 유지보수성과 성능 균형을 좋게 가져가는 데 있다.

### 4-5) 이 프로젝트 기준 연결
- `App.tsx`의 JSX는 React 내부에서 가상 DOM 구조로 다뤄진다.
- `main.tsx`의 렌더링 과정에서 React가 계산한 변경 사항만 `#root` 아래 실제 DOM에 반영한다.

## 5. Component와 Props

### 5-1) Component란?
- Component는 UI를 독립적인 단위로 나눈 함수(또는 클래스)다.
- React에서는 보통 함수형 컴포넌트를 사용하며, JSX를 반환해 화면을 구성한다.
- 이 프로젝트의 `App.tsx`는 루트 Component다.

간단히 말해, Component는 화면을 조립하는 "부품"이다.

### 5-2) Props란?
- Props는 부모 Component가 자식 Component에게 전달하는 읽기 전용 데이터다.
- 자식은 받은 props를 기반으로 화면을 그리지만, props 자체를 직접 수정하지는 않는다.
- 같은 Component라도 다른 props를 받으면 다른 UI를 렌더링할 수 있다.

간단히 말해, Props는 부품에 넘겨주는 "설정값"이다.

### 5-3) 짧은 예시
```tsx
type TitleProps = {
	text: string
}

function Title({ text }: TitleProps) {
	return <h2>{text}</h2>
}

function App() {
	return <Title text="Vite + React" />
}
```

- `Title`은 Component
- `text`는 Props
- 부모(`App`)가 자식(`Title`)에게 값을 전달한다.

### 5-4) 현재 문서 흐름과 연결
1. `main.tsx`가 `<App />` Component를 루트에 렌더링한다.
2. `App` 내부에서 다른 Component들을 조합해 화면을 만든다.
3. 필요한 데이터는 Props로 전달되고, 값이 바뀌면 가상 DOM 비교 후 필요한 부분만 갱신된다.

## 6. 한 줄 요약
`index.html`이 `main.tsx`를 실행하고, `main.tsx`가 `#root`에 `<App />`를 렌더링하며, App은 여러 Component를 조합하고 Props로 데이터를 전달해 UI를 만든다.
